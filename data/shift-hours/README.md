# Часы смен — локальный справочник (маршруты)

Данные для **построения маршрутов по поездам** (`shiftDetails` / `depotShiftTemplates`).

**Не относится к графикам работы** (`employees`, лист «График работы_Прил») — они по-прежнему из Google API.

## Что лежит в `app/data/`

| Путь | Назначение |
|------|------------|
| `shift-templates.json` | Локальная база часов смен (экспорт из `Часы смен/*.xlsx`) |
| `shift-hours/*.json` | Справочники и схемы для экспорта из xlsx |

## Источник xlsx (не в GitHub Pages)

Нормативные файлы хранятся в корне репозитория: `Часы смен/*.xlsx`.

| Период | Папка | Действует |
|--------|-------|-----------|
| с 24.06.2026 | `Часы смен/*.xlsx` | до 16.07.2026 включительно |
| с 17.07.2026 | `Часы смен/c 17_07/*.xlsx` | с 17.07.2026 |

Экспорт: `npm run export:shifts` → `app/data/shift-templates.json`.

## Формат строки (flat row)

Совпадает с `getShiftDetailsData()` в `google-script/Код.js`:

```json
{
  "date": "пн-чт",
  "route": "Н60",
  "startPlace": "МЯ",
  "startTime": "18:15",
  "trains": "6856 - 6859",
  "endTime": "1:00",
  "nightHours": "3:00",
  "workHours": "6:45",
  "morningRoute": "61У",
  "lunch": "**"
}
```

## Поиск в приложении

`findShiftTemplate(templates, route, dateKey)`:

1. Точная дата (`2026-07-15`)
2. Маркер дня недели (`пн-чт`, `пт`, `сб`, `вс`)
3. Fallback — любая строка с тем же маршрутом

## Ночная смена + утро

- `morningRoute` в строке **Нxx** (col 21 xlsx) → обычно `61У`
- Поезда утра — **отдельная** строка `route: "61У"` с тем же `date`
- `dateKey` утра = **тот же** день, что и ночь (не следующий календарный)
- **Обед у утренних маршрутов (xxУ) нет** — перерыв только у ночи (cols 19–20) и дня (cols 7–8)

## Сегмент 60У (col 22–29)

Часть норматива по **времени** (после обеда). Для **навигации по поездам** не используется — там берётся каталог `61У`.

См. `examples/n60-all-days.json`.

## Google API

На текущем этапе Google **не отключается**. Правила слияния — `merge-rules.json`.
