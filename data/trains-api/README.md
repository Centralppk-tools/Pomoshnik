# Запросы поездов (Яндекс.Rasp) — ОБЯЗАТЕЛЬНО К ПРОЧТЕНИЮ

> **Лимит: ~500 запросов в сутки.** Любая правка логики поездов в `app/index.html`, `trains-local.json`, `trains-uids.json` — **сначала прочитай этот файл целиком.**

---

## Три единственных триггера API

Запрос к Яндексу **только** по явному действию пользователя:

| # | Действие | Функция |
|---|----------|---------|
| 1 | Ввод номеров + **«Показать маршруты»** | `getRouteData()` |
| 2 | **«Маршрут»** у поездов смены (календарь / профиль) | `navigateToRouteWithTrains()` → `getRouteData()` |
| 3 | **«Обновить»** на вкладке маршрута | `refreshRouteData()` → `getRouteData({ threadsOnly: true })` |

**Запрещено** вызывать fetch из:

- `setInterval` / секундомера (`tickScheduleLiveClock`)
- `visibilitychange` / `pageshow`
- полночи (00:00) — только смена «рабочей даты» в UI
- смены вкладок, свайпов, рендера календаря
- chip смены «84 У» (только выбор дня, без API)

Chip «84 У 6:30» **не** грузит поезда. API — только кнопка «Маршрут» у блока «Пoезда» в конструкторе смены.

## Порядок поиска (cache-first)

Для каждого поезда и **конкретной даты** `YYYY-MM-DD`:

1. **`trainThreadsCache`** в localStorage — ключ `номер@дата`. Есть запись → **0 API**.
2. **`threadUid`** — без schedule:
   - `trains-local.json` → поле `threadUid`
   - `data/trains-uids.json` → `uids["6601"]`
   - индекс табло в памяти (если табло на эту дату уже качали)
   - uid из старой записи кэша нитки  
   → один запрос **`/thread/`** (1 API).
3. **Табло Ярославского** (`s2000002`) на дату — **один раз на дату**, не на поезд:
   - все параллельные поиски ждут один `stationScheduleLoaders[station_date]`
   - из табло берётся uid → **`/thread/`** (1 API на поезд).
4. **Fallback:** табло Пушкино (`s9600701`) — **только** если на Ярославском нет номера.
5. **Офлайн-резерв:** `trains-local.json` без uid (2 остановки, без API).

## Сколько стоит один сценарий

| Сценарий | Schedule (табло) | Thread (нитка) | Итого |
|----------|------------------|----------------|-------|
| 3 поезда, uid в базе | 0 | 3 | **3** |
| 3 поезда, табло уже на дату | 0 | 3 | **3** |
| 3 поезда, первый раз на дату | 1–3 страницы | 3 | **4–6** |
| Ночь, 5 поездов, 2 даты, табло обе даты | 2–6 | 5 | **7–11** |
| «Обновить» (threadsOnly) | 0 | N поездов | **N** |

**Нельзя:** 2 станции × 5 страниц × N поездов × 3 даты (старый антипаттерн).

## Кэши (не сбрасывать без причины)

| Кэш | Ключ | Когда сбрасывать |
|-----|------|------------------|
| Нитка поезда | `trainThreadsCache[num@date]` | «Обновить», смена даты с force, «Очистить смену» |
| Табло станции | `stationScheduleCache.boards[date]` (до 4 дат, Ярославский) | «Очистить смену», full refresh по датам смены |
| Индекс uid на дату | `stationUidIndexByDate[date]` | вместе с табло |

**«Обновить»** — два режима:

| Жест | Действие | API |
|------|----------|-----|
| Короткое нажатие | Только нитки (`threadsOnly`) | N × `/thread/` |
| **Удержание ~0,6 с** | Полное: табло D (+ D+1 ночью) + нитки | schedule + thread |

Полное обновление сбрасывает табло **только для дат текущей смены**, не весь кэш приложения.

## Ночная смена и 00:00

- При открытии маршрута с «Ночь» предзагружаются табло на **D** и **D+1** (один раз каждое).
- План поездов (`trainPlan`: номер + `scheduleDate` + `night`/`morning`) строится из шаблона смены и передаётся в `getRouteData` — без тройного smart-search на поезд.
- **Н + У = общий список:** номера из N и U объединяются без повторов (напр. ночь `6256`, утро `6257`/`6258` → `6256 - 6257 - 6258`); ночные ищутся на D, утренние на D+1.
- **Обе даты табло сохраняются** в `localStorage` (`stationScheduleCache.boards`) — после перезагрузки приложения повторный schedule не нужен.
- Секундомер при «Ночь» показывает **рабочую дату** всю смену: `ЧЧ:ММ:СС · ДД.ММ.ГГГГ` — до 00:00 это **D**, после полуночи **D+1** (`getEffectiveShiftLiveDateIso`). **API не вызывается.**
- Та же рабочая дата используется для подсветки карточек и push-уведомлений.
- **`isScheduleContextActive`** — смена «активна» и в D, и в D+1 при включённой «Ночь».
- **API в 00:00 не вызывается.** Данные уже в кэше.

## Файлы

| Файл | Назначение |
|------|------------|
| `app/index.html` | `searchTrainByNumber`, кэши, триггеры |
| `app/trains-local.json` | Резерв + `threadUid` для частых поездов |
| `app/data/trains-uids.json` | Массовый индекс номер→uid (генерация ниже) |
| `tools/enrich-train-uids.py` | **1–3 schedule-запроса** → заполнение uids |

Обновление uid-индекса (когда лимит позволяет):

```bash
npm run enrich:train-uids
# или на конкретную дату:
python tools/enrich-train-uids.py --date 2026-07-15
```
